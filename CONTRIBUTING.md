# NeuralVault Contribution Guide

## Getting Started

1. **Fork the repository** to your GitHub account
2. **Clone your fork** locally: `git clone https://github.com/yourusername/NeuralVault.git`
3. **Create a new branch** for your feature: `git checkout -b feature/your-feature-name`

## Adding New Resources

### For PDF Notes:
1. Name PDFs consistently: `SUBJECT-Unit#.pdf`
2. Place in appropriate folder: `public/resources/[Year]/`
3. Update corresponding JSON file in `public/info/`

### For Updates:
1. Make your changes
2. Test locally in your browser
3. Commit with clear messages: `git commit -m "Add [Subject] Unit 3 notes"`
4. Push to your branch: `git push origin feature/your-feature-name`
5. Create a Pull Request with description

## Coding Standards

- Use meaningful variable names
- Add comments for complex logic
- Test cross-browser compatibility
- Ensure responsive design

## Community Guidelines

- Be respectful and helpful
- Provide context in PRs
- Review others'' contributions
- Maintain code quality

## Questions or Issues?

Open an issue with:
- Clear title
- Detailed description
- Steps to reproduce (if applicable)
- Expected vs actual behavior

Thank you for contributing to NeuralVault!
